===Действия с файлами===

Вы — учитель математики, организующий материалы для классов.

Все действия выполняются в домашнем каталоге пользователя linuxoid.

1. Выведите имя учётной записи текущего пользователя (ожидаемый вывод — linuxoid)
2. Выведите полный путь к текущему рабочему каталогу (ожидаемый вывод — /home/linuxoid)
3. Выведите список файлов и каталогов в текущем каталоге (в списке должен присутствовать каталог worklab)

4. Перейдите в каталог ~/worklab/mathTeacher и выведите список файлов и каталогов, находящихся в нём
5. Перейдите в каталог class_5A
6. Внутри class_5A создайте два подкаталога: lessons и homework.

7. Перейдите в каталог lessons и создайте три файла с содержимым (без кавычек):
    lesson_addition.txt — содержит строку "2 + 3 = 5"
    lesson_subtraction.txt — содержит строку "5 - 2 = 3"
    lesson_multiplication.txt — содержит строку "3 * 4 = 12"
8. Выведите на экран содержимое каждого файла,
чтобы убедиться, что всё записано верно.

9. Вернитесь в каталог class_5A
10. Перейдите в каталог homework и создайте три файла с домашними заданиями (без кавычек):
    hw_addition.txt — содержит строку "1+1, 4+5"
    hw_subtraction.txt — содержит строку "10-3, 7-2"
    hw_multiplication.txt — содержит строку "2*2, 5*3"
11. Выведите на экран содержимое каждого файла, чтобы убедиться, что всё записано верно.

12. Объедините содержимое всех файлов из каталога homework в один общий файл all_homework.txt (он должен находиться в том же каталоге homework).
13. В конец файла all_homework.txt допишите строку "for 5B" (без кавычек).
14. Выведите содержимое файла all_homework.txt, чтобы убедиться, что текст добавлен.

15. Вернитесь в каталог ~/worklab/mathTeacher.
16. Скопируйте подкаталог lessons со всем содержимым из class_5A в каталог class_5B
17. Создайте каталог ~/worklab/mathTeacher/class_5B/homework
18. Переместите файл all_homework.txt из  ~/worklab/mathTeacher/class_5A/homework в  ~/worklab/mathTeacher/class_5B/homework
19. Отредактируйте файл class_5B/homework/all_homework.txt: замените последнюю строку на for 5B from 5A

20. Переместите файл ~/worklab/mathTeacher/5a.jpg в каталог class_5A.
21. Найдите все файлы с расширением .txt в каталоге class_5A/homework и выведете результат в файл ~/worklab/mathTeacher/found_files.txt.
22. Найдите все файлы с расширением .txt в каталоге class_5B/homework и добавьте результат (не перезаписывая!) в тот же файл found_files.txt.
23. Удалите файл 1 сентября.png, если он существует.
24. Удалите каталог class_7A_2011 вместе со всеми его файлами.

    .exam.txt — только чтение для всех (-r--r--r--)
    vpr.txt — чтение и запись для владельца и группы (-rw-rw----)
    grades — только для владельца (drw-------)

22. Создайте файл greeting.sh в ~/worklab/mathTeacher/answers
23. Запишите в файл greeting.sh строку 
    'echo "Hello, Teacher!"'
24. Попробуйте выполнить файл с помощью команды ./greeting.sh (должно быть сообщение «Permission denied»)
25. Добавьте владельцу право выполнения для greeting.sh
26. Снова выполните ./greeting.sh
27. Выведите рекурсивный список всех файлов и каталогов с правами, владельцами, размерами и датой. (обратите внимание на столбцы: inode, права, владелец, группа, размер, дата, имя)

===Жесткие и символьные ссылки===
1. Перейдите в каталог ~/worklab/linksInfo
2. Создайте файл message.txt с содержимым "Экзамен будет 22.07" без ковычек
3. Создайте подкаталог ~/worklab/linksInfo/archive
4. Внутри каталога archive создайте жёсткую ссылку на message.txt с именем exam-date.txt 
5. Рекурсивно выведите список всех файлов/каталогов в linksInfo с подробной информацией (включая inode)
    Обратите внимание:
    — у message.txt и archive/exam-date.txt должны быть одинаковые номера inode.
6. Измените содержимое файла archive/exam-date.txt на "Экзамен будет 22.07.2020" без ковычек
7. Проверьте содержимое исходного файла message.txt 
    Должен отразиться тот же текст, потому что жёсткие ссылки указывают на один и тот же физический файл.
8. В каталоге archive создайте символьную ссылку на message.txt с именем message-sym.txt
9. В каталоге archive создайте символьную ссылку на archive/exam-date.txt с именем exam-date-sym.txt
10. Рекурсивно выведите список файлов/каталогов в linksInfo с inode
    Обратите внимание:
    — символьные ссылки имеют собственные inode, отличные от исходных файлов.
    — они лишь указывают путь к файлу, а не являются копией данных.
11. Удалите исходный файл message.txt
12. Ещё раз выведите список файлов/каталогов в linksInfo с inode
13. Проверьте содержимое жёсткой ссылки archive/exam-date.txt
    Файл должен остаться доступен, потому что жёсткая ссылка хранит те же данные.
14. Проверьте содержимое символьной ссылки archive/exam-date-sym.txt
    Если исходный путь больше не существует — появится ошибка, что файл не найден.

Команды Linux

1. Навигация по файловой системе

Вывести имя пользователя

whoami

Синтаксис:

whoami [ПАРАМЕТРЫ]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: whoami --help

Пример использования:

whoami

Вывести полный путь текущего каталога

pwd

Синтаксис:

pwd [ПАРАМЕТРЫ]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: pwd --help

Пример использования:

pwd

Сменить текущий каталог на указанный

cd

Синтаксис:

cd [ПАРАМЕТРЫ] [ПУТЬ]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: cd --help

[ПУТЬ] — путь к каталогу, который нужно сделать текущим.

Возможные значения [ПУТЬ]:

  • ~ — домашний каталог пользователя
  • .. — каталог уровнем выше
  • /path/to/catalog — абсолютный путь
  • path/to/catalog — относительный путь

Примеры использования:

cd ~
cd ..
cd /etc
cd path/to/catalog

Вывести список файлов и каталогов

ls

Синтаксис:

ls [ПАРАМЕТРЫ] [ФАЙЛ]

Возможные значения [ПАРАМЕТРЫ]:

  • -a, --all — показывать скрытые файлы
  • -i, --inode — показывать inode
  • -l — подробный вывод
  • -R, --recursive — рекурсивный вывод

Здесь приведены не все параметры. Подробнее: ls --help

[ФАЙЛ] — путь к файлу или каталогу, содержимое которого нужно отобразить. По умолчанию выводится содержимое текущего каталога.

Примеры использования:

ls
ls -al
ls -liaR /path/to/catalog

2. Основные действия с файлами и каталогами

Создание

Создать каталог

mkdir

Синтаксис:

mkdir [ПАРАМЕТРЫ] КАТАЛОГ

Возможные значения [ПАРАМЕТРЫ]:

  • -p, --parents — не выдавать ошибку, если существует; создавать родительские каталоги при необходимости

Здесь приведены не все параметры. Подробнее: mkdir --help

КАТАЛОГ — путь к каталогу, который нужно создать.

Примеры использования:

    mkdir mycatalog
    mkdir -p ~/test/mycatalog

Создать файл

touch

Синтаксис:

touch [ПАРАМЕТРЫ] ФАЙЛ

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: touch --help

ФАЙЛ — имя файла, который нужно создать. Также можно прописывать и полное имя файла.

Примеры использования:

    touch myfile
    touch path/to/catalog/myfile

Редактирование

Вывести текст на экран

echo

Синтаксис:

echo [ПАРАМЕТРЫ] [ТЕКСТ]

Возможные значения [ПАРАМЕТРЫ]:

  • -n — не добавлять в конец вывода перевод на новую строку.
  • -e — включить интерпретацию управляющих последовательностей (\n, \t)

Здесь приведены не все параметры. Подробнее: info echo (для выхода нажмите q)

[ТЕКСТ] — непосредственно та информация в текстовом виде, которую нужно вывести.

Пример использования:

echo "Hello"

Вывести содержимое файла

cat

Синтаксис:

cat [ПАРАМЕТРЫ] [ФАЙЛ]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: cat --help

[ФАЙЛ] — путь к файлу, содержимое которого нужно вывести.

Примеры использования:

cat file.txt
cat a.txt >> b.txt

Постраничный просмотр файлов

less

Синтаксис:

less [ПАРАМЕТРЫ] ФАЙЛ

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: less --help

ФАЙЛ — путь к файлу, содержимое которого нужно вывести.

Пример использования:

less path/to/catalog/myfile.txt

Консольный текстовый редактор nano

nano

nano — консольный текстовый редактор, который подходит для редактирования конфигурационных файлов, написания скриптов.

Синтаксис:

nano [ПАРАМЕТРЫ] ФАЙЛ

Возможные значения [ПАРАМЕТРЫ]:

  • -i — включает автоотступы
  • -l — включает номера строк
  • -v — открывает файл только чтение

Здесь приведены не все параметры. Подробнее: nano --help

ФАЙЛ — путь к файлу, с которым нужно работать (отредактировать, просмотреть и т. д.).

Примеры использования:

nano file.txt
nano /etc/hosts
Горячие клавиши
Передвижение
  • ← / → — перемещение по символам влево и вправо
  • ↑ / ↓ — переход между строками
  • Ctrl + A — переход в начало текущей строки
  • Ctrl + E — переход в конец текущей строки
  • Ctrl + Y — прокрутка страницы вверх
  • Ctrl + V — прокрутка страницы вниз
  • Ctrl + C — показать позицию курсора (строка, столбец, процент файла)
Редактирование
  • Ctrl + K — вырезать текущую строку
  • Ctrl + U — вставить вырезанный текст
  • Ctrl + J — выравнивание текста (форматирование абзаца)
  • Ctrl + T — вызов справки и дополнительных команд поиска
Сохранение
  • Ctrl + O — сохранить файл (Nano предложит имя файла)
Выход
  • Ctrl + X — выйти из редактора (при изменениях будет запрошено подтверждение)

Перенаправление вывода

Перенаправить вывод в файл с перезаписью

>

Синтаксис:

КОМАНДА > ФАЙЛ

КОМАНДА — любая команда, которая в результате выводит текстовую информацию (например, cat или echo).

ФАЙЛ — путь к файлу, в который нужно вывести информацию. Если файла нет — он будет создан.

Пример использования:

echo "Hello" > file.txt

Добавить вывод в конец файла

>>

Синтаксис:

КОМАНДА >> ФАЙЛ

КОМАНДА — любая команда, которая в результате выводит текстовую информацию (например, cat или echo).

>> — выходные данные записываются в конец файла. Существующие содержимое файла не стирается. Если файла нет — он будет создан.

[ФАЙЛ] — путь к файлу, в который нужно вывести информацию. Если файла нет — он будет создан.

Пример использования:

echo "Hello" >> file.txt

Скопировать файл или каталог

ИСТОЧНИК — путь к файлу или каталогу, который нужно скопировать.

НАЗНАЧЕНИЕ — путь к каталогу, в который нужно скопировать данные.

Примеры использования:

Переместить/переименовать файл или каталог

ИСТОЧНИК — путь к файлу или каталогу, который нужно переместить.

НАЗНАЧЕНИЕ — путь к каталогу, в который нужно переместить данные.

Поиск

Найти файлы по заданным условиям

find

find — утилита для поиска файлов и директорий в файловой системе. Позволяет рекурсивно обходить дерево каталогов, начиная с указанных точек, и находить объекты на основе разнообразных критериев: имени, размера, типа, времени модификации, прав доступа и многих других.

Синтаксис:

find [КАТАЛОГ] [ПАРАМЕТРЫ] [КРИТЕРИЙ_ПОИСКА] [ПАТТЕРН] [ДЕЙСТВИЕ]

[КАТАЛОГ]: один или несколько каталогов, с которых начинается поиск (например, . — текущий каталог, / — корень файловой системы). Если каталог не указан, по умолчанию используется текущий каталог.

Возможные значения [ПАРАМЕТРЫ] можно посмотреть в терминале, введя: find --help

Возможные значения [КРИТЕРИЙ_ПОИСКА]:

Критерий Описание Пример
-name pattern Имя файла соответствует шаблону (с учётом регистра, shell-глобы: *, ?) find . -name "*.txt"
-path pattern Полный путь соответствует шаблону find /home -path "*/docs/*"
-type t Тип файла:
f — файл,
d — каталог,
l — символическая ссылка,
b — блочное устройство и т.д. 		find . -type f
-size n[ckMG] Размер файла:
c — байты,
k — KiB,
M — MiB,
G — GiB;
+n — больше n,
-n — меньше n 		find . -size +100M
-mtime n Время модификации в днях:
+n — старше n дней,
-n — моложе n дней 		find . -mtime +30
-mmin n Время модификации в минутах find . -mmin -60
-perm mode Права доступа:
точное совпадение (644) или
/mode — любые из битов 		find . -perm 777
-user name Владелец файла find / -user root
-group name Группа файла find . -group users
-empty Пустой файл или каталог find . -empty

Возможные значения [ПАТТЕРН] и [ДЕЙСТВИЕ] можно посмотреть в терминале, введя: find --help

Примеры использования:

find . -type f -name "*.txt"
find . -type f -name "*.txt" > mydir/report.txt

Во втором примере результат работы команды find перенаправляется в файл mydir/report.txt.

Найти строки в файле по шаблону

grep

grep — утилита командной строки, предназначенная для поиска строк, соответствующих заданному шаблону, в одном или нескольких файлах.

Синтаксис:

grep [ПАРАМЕТРЫ] ШАБЛОН [ПУТЬ]

Возможные значения [ПАРАМЕТРЫ]:

  • -i — поиск строки без учёта регистра в указанном файле.
  • -c — нахождение количества строк, которые соответствуют заданному шаблону.
  • -l — отображение файлов, которые содержат заданную строку или шаблон.

Здесь приведены не все параметры. Подробнее: grep --help

ШАБЛОН — строка или регулярное выражение, которое нужно найти.

[ПУТЬ] — имя одного или нескольких файлов, в которых необходимо выполнить поиск. Если указана папка, поиск выполняется рекурсивно.

Пример использования:

grep "python" notes.txt
# найдёт строки с подстрокой "python" в файле notes.txt

Удаление

Удалить файл и каталог

rm

Синтаксис:

rm [ПАРАМЕТРЫ] ФАЙЛ

Возможные значения [ПАРАМЕТРЫ]:

  • -f — игнорировать ошибки и не запрашивать подтверждение удаления файлов.
  • -r — рекурсивное удаление каталогов и их содержимого.
  • -i — запрашивать подтверждение перед удалением каждого файла.
  • -d — удалять пустые каталоги.
  • -v — выводить информацию об удаляемых файлах.

Здесь приведены не все параметры. Подробнее: rm --help

ФАЙЛ — путь к файлу или каталогу, который нужно удатить.

Пример использования:

rm -rf testdir

3. Права доступа

Изменить права доступа

chmod

Синтаксис:

chmod [ПАРАМЕТРЫ] ПРАВА ФАЙЛЫ

Возможные значения [ПАРАМЕТРЫ]:

  • -R, --recursive — рекурсивно изменять файлы и каталоги
  • -f, --silent, --quiet — не выводить большинство сообщений об ошибках
  • -v, --verbose — выводить диагностические сообщения по каждому файлу

Здесь приведены не все параметры. Подробнее: chmod --help

ПРАВА — новые права доступа, которые нужно установить для указанных файлов или каталогов.

ФАЙЛЫ — список файлов или каталогов, для которых нужно изменить права доступа.

Обычный владелец файла может менять права на своих объектах. Для изменения прав чужих файлов команду нужно выполнять от имени суперпользователя (команда sudo)

Примеры использования:

chmod u+rwx filename.py
chmod 674 ~/directory

4. Пользователи и группы

Сменить пользователя

su

Синтаксис:

su [ПАРАМЕТРЫ] [-] [ИМЯ]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть в терминале, введя: su --help

  • - — запуск нового сеанса с окружением, как если бы пользователь только что вошёл в систему под этим пользователем.

Здесь приведены не все параметры. Подробнее: su --help

[ИМЯ] — имя пользователя, от имени которого будут выполняться команды.

Пример использования:

su student

Выполнить команду с правами суперпользователя

sudo

Синтаксис:

sudo [ПАРАМЕТРЫ] [КОМАНДА]

Возможные значения [ПАРАМЕТРЫ] можно посмотреть в терминале, введя: su --help

[КОМАНДА] — любая команда, кторая должна быть выполнена с правами суперпользователя

Пример использования:

sudo nano /etc/hosts

Изменить группу для файлов и каталогов

chgrp

Синтаксис:

chgrp [ПАРАМЕТРЫ] ГРУППА ФАЙЛ

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: chgrp --help

ГРУППА — название новой группы, которая будет назначена файлу.

ФАЙЛ — имя файла, для которого меняется группа.

Обычный пользователь может менять группу только на ту, в которой состоит, и только для своих файлов. Для произвольных изменений команду нужно выполнять от имени суперпользователя (команда sudo)

Пример использования:

sudo chgrp teacher file.txt

Изменить владельца и группу для файлов и каталогов

chown

Синтаксис:

chown [ПАРАМЕТРЫ] [ИМЯ][:[ГРУППА]] ПУТЬ

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: chown --help

[ИМЯ] — имя пользователя, которого нужно назначить владельцем

[:ГРУППА] — название группы, которую нужно назначить файлу

[ПУТЬ] — путь к файлу, для которого меняется владелец или группа

Некоторые особенности синтаксиса:

  • chown user file — изменяет владельца файла или директории на указанного пользователя.
  • chown user:group file — изменяет владельца и группу файла или директории.
  • chown :group file — изменяет только группу файла или директории.
бычный пользователь не может менять владельца (даже своих файлов на другого пользователя). Команду нужно выполнять от имени суперпользователя (команда sudo)

Пример использования:

sudo chown user:group /path/to/file 
# user новый владелец файла, а group — группа, которой этот файл будет принадлежать.
sudo chown user /path/to/file 
# такое изменение изменит владельца файла на user. Группа файла останется прежней

Изменить атрибуты учётной записи пользователя

usermod

Синтаксис:

usermod [ПАРАМЕТРЫ] ИМЯ

Возможные значения [ПАРАМЕТРЫ]:

  • -l — изменение имени пользователя
  • -g — добавление пользователя в группу

Здесь приведены не все параметры. Подробнее: usermod --help

[ИМЯ] — имя пользователя, для которого применяются изменения.

Модификация учётной записи пользователя (например, смена группы, UID и т.д.). Почти всегда выполяется от имени суперпользователя (команда sudo)

Примеры использования:

sudo usermod -l newusername oldusername
sudo usermod -g newgroupname username

Управление паролями учётных записей

passwd

Назначение команды:
Управляет паролями пользователей. Позволяет:

  • изменять пароль текущего пользователя;
  • изменять пароль другого пользователя (для администраторов);
  • блокировать пароль;
  • задавать срок действия пароля.

Синтаксис:

passwd [ПАРАМЕТРЫ] [ИМЯ]

Возможные значения [ПАРАМЕТРЫ]:

  • -d, --delete — удалить пароль пользователя
  • -l, --lock — заблокировать пароль пользователя
  • -n, --mindays MIN_DAYS — минимальное число дней между сменами пароля

Здесь приведены не все параметры. Подробнее: passwd --help

[ИМЯ] — имя пользователя, для которого выполняются действия.

Обычный пользователь может изменить только свой пароль. Для смены пароля другого пользователя команду нужно выполнять от имени суперпользователя (команда sudo)

Пример использования:

passwd user

Добавление нового пользователя

adduser

Назначение команды:
Создаёт нового пользователя и автоматически настраивает его окружение (домашний каталог, группу, пароль).

Синтаксис:

adduser [ПАРАМЕТРЫ] ИМЯ

Возможные значения [ПАРАМЕТРЫ]:

  • --home DIR — указать домашнюю директорию пользователя
  • --no-create-home — не создавать домашний каталог
  • --disabled-login — запретить вход пользователя

Здесь приведены не все параметры. Подробнее: adduser --help

[ИМЯ] — имя создаваемого пользователя.

Создание нового пользователя всегда выполяется от имени суперпользователя (команда sudo)

Пример использования:

sudo adduser testuser

Добавление новой группы

groupadd

Синтаксис:

groupadd [ПАРАМЕТРЫ] ГРУППА

Возможные значения [ПАРАМЕТРЫ]:

  • -p ПАРОЛЬ, --password ПАРОЛЬ — применяет пароль для новой группы
  • -U ПОЛЬЗОВАТЕЛИ, --users ПОЛЬЗОВАТЕЛИ — задаёт список пользователей-членов группы
  • -f, --force — не выдаёт ошибку, если группа уже существует

Здесь приведены не все параметры. Подробнее: groupadd --help

ГРУППА — имя создаваемой группы.

Создание новой группы всегда выполяется от имени суперпользователя (команда sudo)

Примеры использования:

sudo groupadd teacher
sudo groupadd teacher -U testuser

Завершить текущий сеанс командной строки

exit

Синтаксис:

exit

5. Управление пакетами

Управление пакетами

apt

Advanced Package Tool (APT) — утилита командной строки для управления программными пакетами в дистрибутивах Linux на базе Debian (например, Ubuntu, Linux Mint).

Синтаксис:

apt КОМАНДА [ПАРАМЕТРЫ]
  • sudo apt update Обновление баз данных пакетов (указанных в/etc/apt/sources.list)
  • sudo apt upgrade Обновление системы
  • sudo apt search ПАКЕТ Поиск пакетов
  • sudo apt search ^ПАКЕТ Поиск пакетов по регулярному выражению
  • sudo apt show ПАКЕТ Поиск пакетов и выводит информацию о пакете
  • sudo apt install ПАКЕТ Установить пакет
  • sudo apt purge ПАКЕТ Удалить пакет и его конфигурационные файлы. Возможно использовать для уже удаленного пакета.
  • sudo apt remove ПАКЕТ Удалить пакет
  • sudo apt autoremove Удаление всех пакетов-сирот
  • sudo apt autoclean Очистка кэша неустановленных пакетов
  • sudo apt clean Очистка кэша пакетов

ПАКЕТ — название программного пакета, для которого требуется выполнить действие.

Более полный список команд можно получить, введя в терминале: apt --help

Примеры использования:

sudo apt update
sudo apt upgrade
sudo apt install geany

Найти путь к исполняемому файлу

which

Синтаксис:

which [ПАРАМЕТРЫ] КОМАНДА

Возможные значения [ПАРАМЕТРЫ] можно посмотреть, введя в терминале: info which (для выхода нажмите q)

КОМАНДА — имя команды или исполняемого файла, который нужно найти.

Пример использования:

which ls
# выводит абсолютный путь к исполняемому файлу команды ls, 
# который обычно находится по адресу /usr/bin/ls

6. Ссылки

Создание жёстких и символьных ссылок

ln

Жёсткие ссылки создаются с помощью команды ln без дополнительных опций. По сути, это копии исходного файла, но они используют один и тот же inode (метаданные файла) и физическое место хранения. Изменения в содержимом файла отражаются во всех жёстких ссылках.

Символьные (мягкие) ссылки создаются с помощью опции -s вместе с командой ln. Это указатели на исходный файл или директорию, они содержат путь к целевому объекту.

Синтаксис:

ln [ПАРАМЕТРЫ] ИСТОЧНИК НАЗНАЧЕНИЕ

Возможные значения [ПАРАМЕТРЫ]:

  • -s — создаёт символическую ссылку
  • -f — аставляет команду перезаписать уже существующий файл

Здесь приведены не все параметры. Подробнее: ln --help

ИСТОЧНИК — файл или каталог, на который делается ссылка.

НАЗНАЧЕНИЕ — место для сохранения ссылки — если это поле не заполнено, символическая ссылка сохраняется в текущем рабочем каталоге.

Пример использования:

ln file link
ln -s file symlink
